# 回滚手册

## 1. 回滚触发条件
- 关键任务失败率显著升高
- 复制中断且短时间无法恢复
- 新版本出现严重功能回归

## 2. 回滚前检查
- 记录当前版本号与提交ID
- 备份 `multi_source_info.json`
- 导出近24小时任务/来源日志

## 3. 回滚步骤
1. 停止正在执行的初始化任务（必要时取消）
2. 切换插件代码到上一个稳定提交
3. 重启相关服务并验证插件可访问
4. 执行关键链路验证：
   - 查看来源列表
   - 查看 channel 状态
   - 触发并验证一条测试任务

## 4. 回滚后验证
- 关键来源状态恢复正常
- 无新增严重错误日志
- 运维和业务方确认恢复

## 5. 事后复盘
- 记录根因、影响范围、改进措施
- 在开发留痕追加回滚事件记录
