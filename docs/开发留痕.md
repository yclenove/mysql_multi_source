## 阶段 6：实战打磨 & 物理模式握手单（已完成）

### 阶段目标

根据实际试用反馈，修复任务调度可靠性、导入进度可观测性、以及物理模式 SSH 的"傻瓜化"配置。

### 主要变更

- **任务执行可靠性**：
  - `trigger_bootstrap_task` 改为「线程主 + 子进程备」双路径：进程内 daemon 线程保证在 BaoTa 进程存活时一定跑起来；子进程作为备份路径把 stdout/stderr 写到 `task_xxx.stderr.log` 避免静默崩溃
  - `run_bootstrap_task` 入口新增 CAS：`pending → running` 原子切换，两路同时抵达时第二个静默退出，不再双执行打架
  - 触发前对 `start_sync.py` / `python_bin` 做存在性预检，缺失直接写日志说明原因
- **导入进度可观测**：
  - `_run_logical_bootstrap` 增加后台 heartbeat 线程（5s/次）即使 `mysqldump`/`mysql` 阻塞也能保持心跳
  - 导入循环定期查 `information_schema.tables` 对比 dump 文件大小，输出 "导入 m1_111111 中 45MB/1471MB (3%) · 已用 0m45s" 这种实时步骤文本
  - 日志仅在 MB 变化 >10 或百分比变化 ≥5 时追加，消除重复行刷屏
  - 前端 step 5 增加真实进度条 + 大库导入提示
- **物理模式 · 握手单**（新）：
  - 后端新增 `replica_generate_ssh_key`（从库幂等生成 ed25519 密钥）
  - 后端新增 `replica_export_handshake`（打包公钥 + replica_ip + hostname + source_id 为 base64 握手单，24h 有效）
  - 后端新增 `master_import_handshake`（同时兼容握手单和裸公钥），幂等写入 `/root/.ssh/authorized_keys`，审计记录来源
  - 后端新增 `replica_test_ssh_to_master`：先做 TCP 22 端口探测（2s 超时），再跑 `ssh BatchMode=yes ConnectTimeout=5`，`UserKnownHostsFile=/dev/null` 避免首次连接卡住
  - 前端从库向导 step 4 新增"启用物理模式（SSH 握手单）"折叠卡，三个按钮：生成握手单 / 复制握手单 / 测试到主库的 SSH
  - 前端主库向导 step 4 + summary view 同时展示"物理模式 · 粘贴握手单"卡片，避免用户找不到
- **仪表盘显示一致性**：
  - 源卡片状态标签新增"初始化中 NN%"/"等待初始化"状态，不再在 bootstrap 期间把 IO/SQL 的 No 显示成红色"异常"
  - "启动"按钮在初始化期间改为"初始化中…"disabled，防止误触重复触发
  - `healthColor` / `healthText` 增加 null-safe 防御，避免单处引用错误把整页炸掉
- **UX 体验**：
  - 仪表盘"查看日志"改为手动触发，不再在页面切换时自动弹出
  - 所有向导步骤切换自动滚动到顶（`scrollPluginTop` utility，兼容 BaoTa iframe/layer 容器）
  - 安装 xtrabackup 增加多阶段回退脚本（Debian/RHEL），自动配置 percona-release 源
  - `master_create_repl_user` 授权覆盖 RELOAD / PROCESS / LOCK TABLES / SHOW VIEW / EVENT / TRIGGER 避免 mysqldump 权限错误

### 自测结果

- 任务从 `pending` 到 `running` 切换可靠，日志持续输出无静默
- 1.5 GB 级别的数据库导入过程中 MB / 百分比 / 已用时 实时刷新
- 握手单从从库生成到主库安装流程走通，`/root/.ssh/authorized_keys` 幂等追加
- `replica_test_ssh_to_master` 最坏情况 10s 内返回，不再长时间阻塞 UI
- 源删除后 pending 任务会被级联取消，不再"复活"

### 已知问题

- 跨机 SSH 握手仍需用户手动复制粘贴握手单文本（这是为了避免给从库直连主库 HTTP 端点打洞；后续若引入 BaoTa 跨节点通道再改为推送）
- 物理模式 SSH 探测假设 root@ 登录；如用户自定义 ssh_user 需后续扩展
