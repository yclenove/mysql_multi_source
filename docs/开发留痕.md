## 阶段 2：真实 channel 操作接入（已完成）

### 阶段目标

- 后端能力升级：

- 前端能力升级：

  - 新增来源连通性检查
  - 新增按 source 的状态弹窗查看

### 自测结果

- `python -m py_compile mysql_multi_source_main.py` 通过
- IDE Lint 检查通过（无错误）
- API 与前端按钮映射关系检查通过

### 已知问题

- 当前仅支持 GTID 模式，尚未实现位点模式兜底
- 尚未实现 source 级数据库映射（`source_db -> target_db`）

## 阶段 7-11：双引擎、可靠任务与hardening交付（已完成）

### 主要变更

  - 新增 `run_physical_bootstrap`（xtrabackup/mariabackup 探测）

- 诊断与观测：

  - 运维文档：`docs/运维手册.md`、`docs/回滚手册.md`、`docs/上线检查清单.md`、`docs/验收报告模板.md`

## 阶段 12：mysqldump 环境实装（已完成）

### 阶段目标

按现场“仅 mysqldump/mysql 可用”条件，将初始化链路落地为可执行模式。

### 主要变更

- 新增 `auto` 模式决策：优先 physical，不满足自动走 logical
- logical 链路改为真实执行：
  - 建目标库（target_db）
  - 执行 mysqldump 导出 source_db
  - 使用 mysql 导入到 target_db
- 前端初始化模式下拉增加 `auto`

### 自测结果

- 语法检查通过
- 逻辑流程静态检查通过

### 已知问题

- 若主库复制账号缺少导出权限，logical 链路会失败并进入重试/失败状态
