# 用户使用手册

## 1. 插件用途

`mysql_multi_source` 用于在一个从库实例上接入多个主库来源，并按 channel 管理同步任务。

适用场景：
- 多主库数据汇聚到一个从库实例
- 大库初始化后接管增量同步
- 需要可视化查看任务、诊断和日志

## 2. 使用前准备

- 从库可访问主库 3306 端口
- 主库已创建复制账号
- 建议主从都开启 GTID
- 若希望更快初始化，建议安装 `xtrabackup/mariabackup`
- 若无物理工具，可使用 `mysqldump/mysql` 逻辑链路

## 3. 页面功能说明

### 3.1 顶部按钮
- `健康检查`：检查插件基础状态
- `主库检测`：输出主库健康检测报告（GTID/binlog/server_id等）
- `主库修复预览`：查看将要变更的修复项
- `主库一键修复`：自动修复主库关键配置（高危，可选自动重启）
- `重启MySQL`：主库修复后可一键重启服务（高危）
- `GTID状态`：查看从库 GTID 是否开启
- `总览指标`：查看来源数量、任务成功/失败等汇总
- `检测工具`：检查是否安装物理/逻辑初始化工具
- `审计日志`：查看关键操作审计记录
- `回滚快照`：查看可回滚快照并回滚
- `刷新来源`：刷新来源列表

### 3.1.1 无需命令行主流程
- 默认所有配置、检测、修复、任务执行均可在 UI 内完成
- 命令行仅在“非root + 自动安装失败 + 需要高级排障”时使用

### 3.2 添加主库来源
填写：
- 来源ID
- 通道名（channel）
- 主库地址和端口
- 复制账号和密码

点击 `添加` 后会创建来源配置。

### 3.3 库映射配置
在「库映射配置」区域：

- 填写来源 ID，可点 `载入已有映射` 从服务端拉取当前配置
- 用表格逐行填写 **主库库名（source_db）** → **从库目标库名（target_db）**，点 `添加一行` 可增加多库映射
- 点击 `保存映射` 提交；高级用户可展开「JSON 预览」查看自动生成的数据（只读）

后端存储与接口仍为 JSON 数组，例如：

```json
[
  {"source_db":"order_db","target_db":"m1_order_db"},
  {"source_db":"user_db","target_db":"m1_user_db"}
]
```

### 3.4 初始化任务
- 选择模式：
  - `auto`：自动优先 physical，不满足则降级 logical
  - `physical`：强制物理链路
  - `logical`：强制逻辑链路
- 点击 `创建任务`
- 将任务ID填入任务ID输入框后可：
  - `后台执行`
  - `取消任务`
  - `任务详情`
  - `任务日志`

### 3.5 来源操作
每条来源可执行：
- `启动` / `停止`
- `连通性`（网络检查）
- `状态`（复制线程和延迟）
- `诊断`（错误分类和建议）
- `日志`（来源日志）
- `删除`

### 3.6 主从协同向导
- 签名文件通道：
  - `导出签名配置` -> `验证配置` -> `导入配置`
- API握手通道：
  - `创建握手` -> `接收握手` -> `握手状态`

### 3.7 主库自动化增强
- `主库自动修复`支持同时创建复制账号（输入 `repl_user/repl_password/replica_host`）
- `主库连接测试向导`可直接测试网络+账号可用性并返回失败分类

### 3.8 压测向导（UI）
- 在页面中直接填写来源数、每来源任务数、模式
- 点击 `开始压测` 后自动创建并触发压测任务
- 点击 `查看压测报告` 查看成功率、平均耗时、失败类型

## 4. 推荐操作流程

1. 主库检测 -> 修复预览 -> 一键修复（自动配置优先）
2. 若修复提示需要，再执行重启MySQL（可选自动重启或手工重启）
3. 添加来源
4. 保存库映射
5. 检测工具并确认初始化模式
6. 创建初始化任务
7. 后台执行任务
8. 查看任务详情和任务日志
9. 执行来源诊断，确认状态正常
10. 启动并观察 channel 同步状态

## 5. 常见问题

### Q1: 提示“请先配置库映射后再创建初始化任务”
先在“库映射配置”里保存 `source_db -> target_db`。

### Q2: 提示没有 root 无法自动安装工具
插件会返回手工安装命令，按提示在 root 终端执行后再检测。

### Q3: 任务失败怎么办
先看 `任务日志` 和 `诊断`，常见为网络、权限、GTID、冲突问题；修复后重试或新建任务。

### Q4: 主库 5.7、从库 8.0 能否使用
可以，但建议 GTID + ROW 模式，并优先做小库验证后再全量切换。
