# 用户使用手册 v2.0

## 1. 插件用途

`mysql_multi_source` 用于在一个从库实例上接入多个主库来源，按 channel 管理同步任务。

适用场景：
- 多主库数据汇聚到一个从库实例
- 大库初始化后接管增量同步
- 需要可视化查看任务、诊断和日志

## 2. 使用前准备

- 从库可访问主库 3306 端口
- 建议主从都开启 GTID
- 若希望更快初始化，建议安装 `xtrabackup/mariabackup`
- 若无物理工具，可使用 `mysqldump/mysql` 逻辑链路

## 3. 首屏四卡片

打开插件后默认看到 4 张任务卡片：

| 卡片 | 场景 |
|------|------|
| **我是主库，帮我准备好** | 在主库服务器运行：检测 → 修复 → 创建复制账号 → 导出配置单 |
| **我是从库，帮我接入主库** | 在从库服务器运行：粘贴配置单 / 手填 → 选库 → 开始同步 |
| **查看复制状态仪表盘** | 日常查看所有来源的同步状态、任务进度 |
| **我的复制有问题** | 一键扫描所有来源和任务，按类展示问题并提供修复建议 |

插件会自动检测当前机器角色并高亮推荐卡片。

## 4. 向导一：帮我成为主库（4 步）

1. **体检**：自动检测 gtid_mode、binlog、server_id 等，绿/黄/红标签
2. **修复预览**：before/after 对比，确认将变更的配置项
3. **一键修复**：可选同时创建复制账号 + 修复后自动重启
4. **导出配置单**：生成签名 JSON 文本（内含地址、账号、密码），可复制粘贴给从库

## 5. 向导二：帮我接入主库（5 步）

1. **填主库信息**：粘贴配置单（推荐）或手填 4 字段（IP、端口、账号、密码）
2. **连通性检查**：三维检查（网络、账号、GTID），全绿即可
3. **选库**：自动列出主库用户库 checkbox，从库目标库名自动生成可修改
4. **初始化策略**：自动推荐，可手选 auto / physical / logical
5. **开始同步**：一键启动 → 跳转仪表盘

## 6. 仪表盘

每个主库一张卡片：
- 健康灯（绿/黄/红）
- IO 线程、SQL 线程、延迟指标
- 正在执行的任务进度条
- 操作按钮：启动 / 暂停 / 移除
- 折叠展开：库映射详情

5 秒自动刷新，任务中 2 秒刷新。

## 7. 诊断中心

按类别分组（网络 / 权限 / GTID / 冲突 / 资源 / 配置 / 其它），每条问题提供：
- 一键修复（可行时）
- 详细日志
- 手动修复指引

## 8. 专家视图

右上角切换按钮，进入后恢复 4 个页签：

| 页签 | 内容 |
|------|------|
| 快速接入 | 手动添加来源、连接测试、库映射 JSON |
| 初始化任务 | 创建/执行/取消/恢复任务、工具安装、压测 |
| 运行监控 | 健康检查、GTID、来源列表、总览指标、诊断快照 |
| 主库运维 | 检测/修复/重启、复制账号、审计日志、快照回滚、签名握手 |

## 9. 密码安全

所有复制密码在 `multi_source_info.json` 中以加密形式存储（Fernet AES-128-CBC 或 XOR 混淆回退），密钥文件 `secret.key` 权限 600。旧版明文密码在首次加载时自动迁移。

## 10. 定时巡检

在 cron 中添加以下条目可自动恢复卡死任务：

```
* * * * * btpython /www/server/panel/plugin/mysql_multi_source/start_sync.py tick
```

## 11. 常见问题

| 现象 | 处理 |
|------|------|
| 连通性检查全红 | 检查安全组/防火墙是否放通主库 3306 |
| 账号可用检查失败 | 复制账号 user@host 限制或密码错误 |
| 初始化任务一直 running | 点「诊断」→ 看任务日志；或等 cron tick 自动恢复 |
| 仪表盘无来源 | 先走向导二接入主库 |
