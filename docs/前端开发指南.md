# 前端开发指南

## 技术栈

- **Vue 3** + Composition API (`<script setup>`)
- **Pinia** 状态管理
- **Naive UI** 组件库
- **Vite** 构建
- **vite-plugin-singlefile** 产出内联单 HTML

## 目录结构

```
frontend/
├── package.json
├── vite.config.ts
├── tsconfig.json
├── env.d.ts
├── index.html            # Vite 入口 HTML
└── src/
    ├── main.ts           # 挂载 Vue + Pinia
    ├── App.vue           # 根组件：NConfigProvider + 路由切换
    ├── api/
    │   └── plugin.ts     # 封装 bt_tools.send / fetch 双模调用
    ├── store/
    │   ├── env.ts        # 环境 / 身份 / 视图切换
    │   └── dashboard.ts  # 仪表盘数据 + 轮询
    ├── components/
    │   ├── AppHeader.vue
    │   └── LandingCards.vue
    └── views/
        ├── WizardMaster.vue
        ├── WizardReplica.vue
        ├── DashboardView.vue
        ├── DiagnoseView.vue
        └── ExpertLayout.vue
```

## 开发

```bash
cd frontend
npm install
npm run dev
```

Vite 默认 http://localhost:5173。因为宝塔 `bt_tools` 不可用，`api/plugin.ts` 自动降级为 `fetch` 调用本地代理（需配置 `vite.config.ts` 的 `server.proxy` 或使用 mock）。

## 构建

```bash
npm run build:only
# 或
npx vite build
```

产物为 `frontend/dist/index.html`（CSS/JS 全部内联）。

用根目录脚本一键完成安装 + 构建 + 复制：

```powershell
powershell -File scripts/build_plugin.ps1
```

```bash
bash scripts/build_plugin.sh
```

构建后 `index.html` 覆盖项目根目录，宝塔安装流程零变化。

## API 调用约定

```ts
import { call, isOk, extractMsg, getMessage } from '@/api/plugin'

const res = await call('wizard_detect_env')
if (isOk(res)) {
  const data = extractMsg(res)   // res.msg
  // ...
} else {
  const errMsg = getMessage(res)  // res.msg.message or res.msg (string)
}
```

所有后端接口统一返回 `{ status: boolean, msg: T }`。`msg` 为 dict 时保证有 `message` 和 `code` 字段。

## 视图切换

使用 `useEnvStore().navigate(view)` 切换视图：

- `'landing'` → 首屏四卡片
- `'wizard_master'` → 主库向导
- `'wizard_replica'` → 从库向导
- `'dashboard'` → 仪表盘
- `'diagnose'` → 诊断中心
- `'expert'` → 专家视图

不使用 vue-router（宝塔插件运行于面板 iframe 内，hash 路由不适合）。

## 添加新组件

1. 在 `src/components/` 或 `src/views/` 新建 `.vue` 文件
2. 在 `App.vue` 注册新视图或在已有视图中 import
3. `npm run build:only` 验证构建无误
4. 运行 `scripts/build_plugin.ps1` 更新根 `index.html`
