# 开发文档

## 1. 项目结构

- `info.json`：插件元信息
- `install.sh`：安装/更新/卸载脚本
- `index.html`：插件前端页面
- `mysql_multi_source_main.py`：后端 API 主入口
- `start_sync.py`：后台任务入口
- `.cursor/rules/`：项目规则
- `docs/`：项目文档

## 2. 本地开发说明

### 2.1 代码仓库
- 仓库地址：`https://github.com/yclenove/mysql_multi_source`
- 本地路径：`H:\aicoding\mysql_multi_source`

### 2.2 宝塔插件目录
- 目标目录：`/www/server/panel/plugin/mysql_multi_source`

### 2.3 导入方式
1. 将插件目录打包为 zip
2. 在宝塔中进入“第三方应用 -> 导入插件”
3. 上传 zip 并安装

## 3. API 约定

- 路径格式：`/plugin?action=a&name=mysql_multi_source&s=<method>`
- 返回格式：`public.returnMsg(status, msg)`
- 契约建议：`msg` 优先使用结构化对象，至少包含 `message` 与 `code` 字段；前端可兼容旧字符串 `msg`
- 当前已实现 API：
  - `health_check`
  - `list_sources`
  - `add_source`
  - `remove_source`
  - `start_channel`（真实 channel SQL）
  - `stop_channel`（真实 channel SQL）
  - `channel_status`（实时状态刷新）
  - `test_source_connection`（主库连通性）
  - `get_gtid_status`（GTID 检查）
  - `set_db_mappings`（保存库映射）
  - `list_db_mappings`（查询库映射）
  - `create_bootstrap_task`（创建初始化任务）
  - `run_bootstrap_task`（执行初始化任务编排）
  - `get_bootstrap_tasks`（查看初始化任务）
  - `get_bootstrap_task`（查看单个任务）
  - `cancel_bootstrap_task`（取消初始化任务）
  - `trigger_bootstrap_task`（后台触发初始化任务）
  - `recover_bootstrap_tasks`（恢复卡死任务）
  - `diagnose_source`（来源诊断）
  - `get_source_logs`（来源日志）
  - `get_task_logs`（任务日志）
  - `overview_metrics`（总览指标）
  - `source_detail`（来源详情）
  - `set_running_mode` / `get_running_mode`（运行角色切换）
  - `master_health_check` / `master_health_report`（主库健康检测）
  - `master_auto_fix_preview` / `master_auto_fix_apply`（主库自动修复）
  - `master_restart_mysql`（主库重启）
  - `master_create_repl_user`（主库创建复制账号）
  - `test_master_connection_direct`（主库连接测试向导）
  - `master_export_signed_profile` / `master_get_profile`（签名配置文件）
  - `replica_verify_profile` / `replica_import_profile`（从库导入签名配置）
  - `master_create_handshake` / `replica_accept_handshake` / `handshake_status`（API握手）
  - `list_change_snapshots` / `rollback_snapshot`（快照与回滚）
  - `list_audit_logs`（审计日志）
  - `get_tool_install_command`（获取安装命令）
  - `run_stress_wizard`（UI压测向导执行）
  - `get_stress_report`（压测报告）

## 4. 开发规范

- 文档主语言为中文
- 高风险操作必须具备前端确认
- 所有 channel 操作必须作用于目标 channel，禁止全局误操作
- 变更后需进行基础自测并更新文档

## 5. 下一步开发重点

1. 将API握手改造成跨主从实例远端传输模式（当前同配置存储示范）
2. 增加更细粒度的权限控制（按角色限制高危操作）
3. 增加可视化任务面板（表格化、分页、过滤）
4. 前端继续拆分 `api/state/render/actions` 为多文件（保持无构建链）
